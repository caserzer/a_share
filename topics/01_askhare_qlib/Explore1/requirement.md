# Explore1 需求说明：2025-12-31 市值 500 亿以上股票池的 Alpha158 + LightGBM 验证

## 1. 背景

当前 `Alpha158 + LightGBM` 初始实验使用 `selected` 股票池，股票数量约 36 只，横截面样本过小；同时 `topk=30` 几乎覆盖整个股票池，导致模型排序能力很难在组合回测中体现。

本轮 Explore1 目标是扩大股票池，重新验证 `Alpha158 + LightGBM` 在更合理横截面上的预测能力和组合表现。

## 2. 目标

构建一个新的股票池：

- 范围：沪深主板 A 股。
- 市值条件：以 `2025-12-31` 为基准日，总市值大于 `500` 亿元人民币。
- 用该股票池重新运行 `Alpha158 + LightGBM` workflow。
- 对比原始 `selected` 小股票池实验，判断模型表现是否改善。

本阶段只定义需求和验证口径，不直接修改代码、配置或运行实验。

## 3. 股票池定义

### 3.1 纳入范围

纳入：

- 上交所主板 A 股。
- 深交所主板 A 股。

暂不纳入：

- 科创板。
- 创业板。
- 北交所。
- B 股。
- ETF、LOF、场内基金、债券、可转债、指数。
- ST、`*ST`、退市整理期股票。

如后续需要覆盖科创板，应单独建立实验分支，避免和本轮结果混在一起。

### 3.2 市值口径

默认使用：

```text
2025-12-31 总市值 > 500 亿元人民币
```

要求：

- 股票池基准日固定为 `2025-12-31`。
- 股票池生成日期需要记录，且需要和基准日区分。
- 市值字段来源需要记录。
- 单位统一为人民币元，筛选阈值为 `50,000,000,000`。
- 若数据源同时提供总市值和流通市值，本轮优先使用总市值。
- 后续可扩展对比：总市值口径 vs 流通市值口径。

### 3.3 股票池时间处理

第一版使用静态股票池：

```text
以 2025-12-31 的总市值筛选股票，然后用于整段回测。
```

报告中必须明确这是静态股票池，存在幸存者偏差和未来函数风险。该口径适合验证扩大股票池后的链路和模型表现变化，不应直接作为严格策略有效性结论。

正式研究版本应升级为动态股票池：

```text
每个调仓日只使用当时已经上市、未退市、未 ST，且当时总市值大于 500 亿的股票。
```

## 4. 数据要求

### 4.1 行情数据

需要为新股票池准备日频数据：

- open
- high
- low
- close
- volume
- amount
- factor

数据区间建议：

```text
2017-01-01 到最新可用交易日
```

### 4.2 数据质量检查

重新生成并检查数据质量报告，至少包括：

- 每只股票起止日期。
- 缺失值数量。
- 重复日期数量。
- 最大单日价格跳变。
- 最大复权因子跳变。
- 是否存在异常成交量或成交额。

重点抽样核验：

- 大市值银行股。
- 大市值白酒股。
- 大市值新能源股。
- 沪深主板大市值股票。
- 复权因子跳变较大的股票。

## 5. Alpha158 + LightGBM 验证口径

### 5.1 特征

使用 Qlib 内置：

```text
Alpha158
```

第一版不修改 Alpha158 特征，先验证扩大股票池后的基础表现。

### 5.2 模型

使用：

```text
LightGBM / LGBModel
```

第一版可沿用当前 `configs/qlib_lightgbm_alpha158.yaml` 中的模型参数，避免同时改变太多变量。

后续再单独做参数搜索。

### 5.3 数据切分

建议保持和当前实验接近的时间切分：

```text
train: 2017-01-01 到 2022-12-31
valid: 2023-01-01 到 2024-12-31
test:  2025-01-01 到最新可回测日期
```

注意：

- 回测结束日需要至少保留下一个交易日的 calendar entry。
- 如果数据截止到 `T`，组合回测结束日通常应早于或等于 `T - 1` 个交易日。

## 6. 回测策略要求

第一版继续使用：

```text
TopkDropoutStrategy
```

但参数需要根据扩大后的股票池重新设置。

建议至少测试：

| 参数 | 候选值 |
| --- | --- |
| topk | 30, 50, 100 |
| n_drop | 3, 5, 10 |

交易设置可先沿用当前配置：

| 项目 | 建议值 |
| --- | --- |
| benchmark | SH000300 |
| initial account | 1,000,000 |
| freq | day |
| deal_price | close |
| limit_threshold | 0.095 |
| open_cost | 0.0005 |
| close_cost | 0.0015 |
| min_cost | 5 |

本轮已排除创业板，第一版可继续使用主板 `10%` 涨跌停近似设置。

## 7. 评估指标

模型层面：

- best iteration
- train l2
- valid l2
- IC
- ICIR
- Rank IC
- Rank ICIR
- 分年度 IC / Rank IC
- 分月份 IC / Rank IC

组合层面：

- benchmark annualized_return
- excess return without cost annualized
- excess return with cost annualized
- information ratio
- max drawdown
- turnover
- cost
- ffr

重点判断：

```text
Rank IC 是否明显高于原始 selected 股票池实验。
扣成本后超额收益是否改善。
不同 topk / n_drop 参数下结果是否稳定。
```

## 8. 对比基线

至少和以下结果对比：

1. 当前 `selected` 股票池的 Alpha158 + LightGBM 结果。
2. 当前简单 20 日动量 TopK 回测结果。
3. 沪深 300 基准指数。
4. `2025-12-31` 静态股票池等权基准。

静态股票池等权基准定义：

```text
在同一只 2025-12-31 市值 500 亿以上沪深主板股票池内，所有可交易股票等权持有，按日或按月再平衡。
```

等权基准需要和 Alpha158 + LightGBM 组合使用相同的：

- 股票池。
- 回测区间。
- 交易成本假设。
- 停牌和涨跌停处理。

如果模型组合跑赢沪深 300，但跑不赢静态股票池等权基准，说明收益可能主要来自股票池本身，而不是模型排序能力。

如果扩大股票池后仍然表现较差，需要进一步判断是：

- Alpha158 特征不适合当前数据。
- 默认短期 label 噪声过大。
- 股票池构造存在偏差。
- 数据复权或成交量字段存在问题。
- 策略参数与股票池规模不匹配。

## 9. 验收标准

本轮 Explore1 完成后，应至少产出：

- 新股票池生成说明。
- 新股票池成分文件。
- 新数据质量报告。
- 新 Alpha158 + LightGBM workflow 配置。
- 新训练与回测结果。
- 和旧实验的对比结论。

文档结论必须明确区分：

```text
链路是否跑通
模型是否具备预测能力
组合是否具备扣成本后的有效性
结果是否可能过拟合
```

## 10. 非目标

本轮不做：

- 自定义 Alpha 特征开发。
- LightGBM 大规模参数搜索。
- 自定义止损止盈策略。
- 动态市值股票池的完整工程实现。
- 实盘交易接口。

这些可以在 Explore1 跑通后，作为后续 Explore2 或 Explore3 的研究内容。
