可以。把两套建议合并后，我会把它重构成一套 **“EMA 趋势状态 + 市场环境过滤 + 趋势质量评分 + 入场触发 + 风险仓位 + 分层退出”** 的系统，而不是简单变成：

```text
EMA + RSI + MACD + ADX + 布林带 + 成交量
```

后者容易变成指标堆叠，信号更晚、参数更多、过拟合更强。整合后的核心思想是：

> **EMA 多头排列只负责告诉你“有趋势状态”；真正决定是否交易的是市场环境、趋势质量、入场位置和风险收益结构。**

---

# 一、合并后的策略总框架

最终系统可以拆成 7 层：

```text
1. 股票池过滤
2. 大盘 / 行业环境过滤
3. 个股 EMA 趋势状态
4. 趋势质量评分
5. 入场触发：突破 or 回踩
6. 仓位与止损
7. 分层退出与利润保护
```

逻辑上应该是：

```text
先判断现在适不适合做趋势
再判断哪个行业有趋势
再判断哪些个股处于趋势状态
再判断这个趋势是不是高质量
最后才决定怎么买、买多少、怎么退出
```

---

# 二、股票池：保留“大市值”，但不要只按市值

你原来的“大市值股票”思路是对的，可以降低小票被游资、流动性、盘口扰动影响的概率。但只看市值不够，建议股票池这样定义：

```text
基础股票池：
- 非 ST
- 上市满 250 个交易日
- 非长期停牌
- 流通市值排名前 300 / 500 / 800
- 过去20日平均成交额 > 某个阈值
- 剔除长期低流动性、频繁涨跌停、财务异常股票
```

不要预设“科技、新能源、创业板一定更好”。更好的方式是：

```text
只做当前强势行业，
而不是固定做高波动行业。
```

高波动行业可能确实更容易出趋势，但也更容易假突破、快速回撤、情绪杀估值。你的初衷是大市值、机构票、低操纵风险，所以行业选择应该基于 **相对强度**，而不是预设题材。

---

# 三、市场环境过滤：这是减少磨损的第一优先级

趋势策略最大的亏损来源通常不是止损单，而是：

```text
震荡市里反复出现“看似多头”的假信号。
```

所以入场前必须先问：

> 当前市场是否适合做多头趋势？

建议用三类过滤器。

## 1. 指数趋势过滤

```text
沪深300 / 中证A500 / 中证800 close > EMA60
指数 EMA60 过去10~20日斜率 > 0
指数 close > EMA120 更好
```

如果指数低于 EMA60，或者 EMA60 斜率向下，停止新开仓，已有仓位只做风控和退出。

## 2. 市场宽度过滤

```text
股票池中 close > EMA60 的股票比例 > 50% 或 55%
股票池中 EMA20 > EMA60 的股票比例 > 40% 或 50%
```

这比单看指数更稳。因为有时候指数被少数权重股拉住，但大多数股票已经不行了。

## 3. 行业趋势过滤

```text
行业指数 close > EMA60
行业 EMA60 斜率 > 0
行业60日收益 > 沪深300 / 中证A500 60日收益
行业内强势股比例 > 50%
```

这样做的效果是：你不再孤立地买个股 EMA 多头，而是在 **指数顺风、行业顺风、个股顺风** 时才买。

---

# 四、EMA 多头排列：从“入场信号”降级为“候选状态”

这是整合后的关键变化。

原策略：

```text
EMA多头排列 → 买入
```

改成：

```text
EMA多头排列 → 加入候选池 → 等待高质量入场触发
```

建议个股候选条件：

```text
EMA20 > EMA60
EMA60 过去10日斜率 > 0
close > EMA60
close 不要距离 EMA20 太远
过去20日波动率不能极端过高
过去20日成交额不能太低
```

其中 “不要距离 EMA 太远” 很重要，否则你会经常追到短期加速末端。

可以用：

```text
(close - EMA20) / close < 2 * ATR20 / close
```

或者更简单：

```text
close / EMA20 - 1 < 5% ~ 8%
```

这个条件会少抓一部分急拉行情，但会明显减少追高假突破。

---

# 五、不要堆指标，要做“趋势质量评分”

另一个人的建议里提到 RSI、MACD、ADX、成交量、布林带，这些可以用，但不要都做成硬条件。

错误方式：

```text
EMA多头
AND RSI > 50
AND MACD金叉
AND ADX > 25
AND 成交量 > 5日均量 * 1.5
AND 布林带突破
```

这样会导致：

```text
信号滞后
交易太少
参数过拟合
回测好看，实盘难做
```

更好的方式是：把它们放进一个 **趋势质量评分 trend_score**。

示例：

```text
trend_score =
  0.25 * EMA60斜率
+ 0.20 * 60日相对强度
+ 0.15 * EMA20 / EMA60 乖离
+ 0.15 * 成交额相对强度
+ 0.10 * MACD / 动量状态
+ 0.10 * ADX / 趋势强度
- 0.15 * 过热惩罚
- 0.10 * 波动率过高惩罚
```

然后只买：

```text
trend_score 排名前 10% / 20% 的候选股
```

这样，RSI、MACD、ADX、成交量、布林带都被吸收了，但不会变成机械的“指标拼盘”。

# 六、指标怎么整合：保留、修改、降权

下面是我对两套建议的合并处理。

| 指标 / 规则  |  是否保留 | 整合方式                      |
| -------- | ----: | ------------------------- |
| EMA 多头排列 |    保留 | 只做候选状态，不直接买               |
| 成交量确认    |    保留 | 判断突破质量，不单独决定买入            |
| ADX      | 保留但降权 | 用于趋势强度评分，不建议 ADX<20 一刀切空仓 |
| MACD     |   可保留 | 作为动量评分，不必强制金叉             |
| RSI      |   可保留 | RSI>50 可加分，RSI 极端过热要扣分    |
| 布林带宽度    | 保留但重写 | 用于识别波动压缩，不是简单禁止交易         |
| ATR 止损   |  强烈保留 | 替代固定百分比止损                 |
| 分批止盈     |  谨慎保留 | 不要过早分批，否则伤害肥尾             |
| EMA60 退出 |    保留 | 作为最后趋势退出，不作为唯一退出          |
| 大市值股票池   |    保留 | 但加流动性、上市天数、非ST、行业过滤       |
| AI 模型    |  后期引入 | 先规则版，再因子版，再模型版            |

# 七、入场规则：分成“突破型”和“回踩型”

你的原策略容易买在假突破上，所以入场应该分两类。

## 方案 A：突破确认型

适合捕捉趋势刚启动。

```text
前置条件：
- 大盘环境 OK
- 行业环境 OK
- 个股 EMA20 > EMA60
- EMA60 斜率向上
- trend_score 位于候选池前 10% / 20%

突破触发：
- close > 过去60日高点
- 成交额 > 过去20日平均成交额 * 1.1 或 1.2
- 收盘价接近当日高位
- 上影线不能太长
- close 距离 EMA20 不能过远
```

防假突破可以加确认：

```text
第1天突破不立刻重仓；
加入观察池；
第2~3天仍站在突破位上方，或回踩突破位不破，再买入。
```

这样会牺牲一点入场速度，但会减少很多“一日游突破”。

---

## 方案 B：趋势回踩型

我更建议你重点测试这个，因为你做的是大市值股。

```text
趋势已经成立：
EMA20 > EMA60
EMA60 上行
行业趋势向上
个股 trend_score 靠前

回踩买点：
价格回踩 EMA20 / EMA30 附近
不跌破 EMA60
回踩过程中缩量
重新站上短 EMA 或出现放量阳线
```

回踩型的优点：

```text
买点更低
止损更近
风险收益比更好
不容易追在短期高潮
```

缺点是：

```text
强趋势急拉时可能不给回踩机会
```

所以最终可以突破型和回踩型都做，但分开统计，不要混在一起。

# 八、止损：固定止损改成 ATR + 结构止损

固定 5%、8%、10% 止损的问题是：

```text
不同股票波动率不同
同一股票不同阶段波动率也不同
```

整合后的初始止损建议：

```text
突破买入：
止损 = 突破平台下沿 - 0.5 * ATR20

回踩买入：
止损 = 回踩低点 - 0.5 * ATR20

通用版本：
止损 = 入场价 - 2 * ATR20
```

更稳的是取两者中更合理的位置：

```text
initial_stop = min(结构低点 - 0.5 * ATR20, 入场价 - 2 * ATR20)
```

不过注意，止损太远会导致仓位变小，止损太近会被洗掉。这个问题要通过仓位管理解决，而不是硬调止损。

---

# 九、仓位：从“固定买入金额”改成“按风险单位建仓”

这是趋势系统非常关键的一步。

建议：

```text
单笔最大亏损 = 总资金的 0.5% ~ 1%
买入股数 = 单笔最大亏损 / (入场价 - 初始止损价)
```

例如：

```text
总资金 = 1,000,000
单笔风险 = 0.5% = 5,000
入场价 = 20
止损价 = 18.8
每股风险 = 1.2
买入股数 ≈ 5,000 / 1.2 = 4,166 股
```

再叠加组合约束：

```text
单票最大权重：3% ~ 6%
单行业最大权重：20% ~ 30%
最大持仓数量：15 ~ 30只
单日最大新增仓位：10% ~ 20%
```

这样可以避免某一行业、某几只票把组合风险全部吃掉。

# 十、退出：EMA60 不要废掉，但不能作为唯一退出

你原来的 “直到回踩 EMA60” 有一个优点：

```text
能抓住大趋势肥尾
```

但缺点也明显：

```text
普通趋势利润回吐太大
肥尾之外的交易利润保护不足
```

整合后的退出应该分层。

## 1. 初始失败退出

```text
触发初始止损，退出。
买入后 8 ~ 12 个交易日仍未盈利，退出或减仓。
买入后快速跌回突破平台，退出。
```

这个规则专门处理：

```text
短期内没有进入上升通道，就撤出。
```

这是你原策略里本来就有的思想，应该保留。

## 2. 盈利保护退出

```text
盈利达到 1R：
止损抬到成本价附近。

盈利达到 2R：
用 EMA20 / 前低 / Chandelier Stop 跟踪。

盈利达到 3R：
如果价格明显远离 EMA20，可部分减仓或收紧止盈。
```

这里的 R 是单笔风险：

```text
R = 入场价 - 初始止损价
```

## 3. 趋势终结退出

```text
剩余仓位跌破 EMA60，最终退出。
```

也就是说：

```text
EMA60 = 最后一档趋势退出
EMA20 / ATR trailing = 利润保护退出
初始止损 = 错误认定退出
时间止损 = 信号失效退出
```

这比 “只等 EMA60” 更完整。

# 十一、分批止盈怎么处理

另一个建议里提到：

```text
盈利1.5倍ATR卖1/3
盈利3倍ATR再卖1/3
剩余仓位用EMA60
```

我建议谨慎使用。原因是你的策略目标本来就是抓肥尾，如果过早分批止盈，可能会变成：

```text
亏损单完整亏
盈利单太早卖
大趋势只剩小仓位
```

更好的分批方式是：

```text
不在固定 1.5ATR 止盈；
只在极端乖离时减仓；
或者在盈利达到 2R / 3R 后，将止损上移，而不是直接卖出。
```

可以测试两个版本：

```text
版本 A：不分批，只用 trailing stop
版本 B：极端乖离时减 1/3，剩余用 trailing stop
```

然后比较：

```text
年化收益
最大回撤
收益回撤比
肥尾交易贡献
平均利润回吐
剔除前5笔大盈利后的收益
```

不要凭感觉决定分批止盈。

# 十二、整合后的完整交易规则示例

可以先用这个作为第一版规则系统。

## 买入前置条件

```text
市场条件：
- 沪深300 / 中证A500 close > EMA60
- 指数 EMA60 斜率 > 0
- 股票池中 close > EMA60 的比例 > 55%

行业条件：
- 行业指数 close > EMA60
- 行业指数 EMA60 斜率 > 0
- 行业60日收益 > 沪深300 60日收益

个股条件：
- 非ST
- 上市满250日
- 流通市值排名前500
- 20日平均成交额达标
- EMA20 > EMA60
- EMA60 斜率 > 0
- close 距离 EMA20 不超过 2 * ATR20
- trend_score 排名前20%
```

## 入场触发

突破型：

```text
close > 过去60日高点
成交额 > 过去20日均值 * 1.1
收盘价位于当日振幅上半区
突破后1~3日未跌回平台
```

回踩型：

```text
EMA20 > EMA60
价格回踩 EMA20 / EMA30 附近
回踩缩量
重新站上 EMA20
或出现放量转强阳线
```

## 止损

```text
初始止损 = 结构低点 - 0.5 * ATR20
或 入场价 - 2 * ATR20
```

## 仓位

```text
单笔风险 = 总资金 0.5% ~ 1%
单票权重上限 = 3% ~ 6%
行业权重上限 = 20% ~ 30%
```

## 退出

```text
亏损退出：
- 跌破初始止损
- 买入后8~12日仍无盈利
- 突破失败跌回平台

盈利保护：
- 盈利 >= 1R，止损抬到成本附近
- 盈利 >= 2R，使用 EMA20 / ATR trailing
- 极端乖离可减仓

趋势终结：
- 剩余仓位跌破 EMA60，清仓
```

# 十三、Qlib 里怎么落地

Qlib 适合把这套系统拆成三层：

```text
因子层：构造 EMA、动量、波动、成交量、相对强度因子
模型层：可选，用 LightGBM 等模型给信号打分
策略层：根据评分、仓位、止损、退出规则生成交易
```

Qlib 支持公式化 alpha，文档里就用 `EMA($close, 12)`、`EMA($close, 26)` 这类表达式构造 MACD，因此你的 EMA 斜率、均线乖离、放量、突破、波动率都可以直接写成表达式因子。([Qlib Documentation][1])

Qlib 也支持股票池和动态过滤。它的文档说明 stock pool 是股票列表及其日期范围，并且提供 `ExpressionDFilter`，可以按表达式过滤标的，比如按截面排名或时间序列条件过滤。([Qlib Documentation][2])

---

## 1. 因子表达式示例

```python
fields = [
    # EMA 状态
    "EMA($close, 20) / EMA($close, 60) - 1",
    "EMA($close, 60) / Ref(EMA($close, 60), 10) - 1",

    # 价格位置
    "$close / EMA($close, 20) - 1",
    "$close / Max(Ref($high, 1), 60) - 1",

    # 动量
    "$close / Ref($close, 20) - 1",
    "$close / Ref($close, 60) - 1",

    # 成交量
    "$volume / Mean(Ref($volume, 1), 20) - 1",

    # 波动率
    "Std($close / Ref($close, 1) - 1, 20)",

    # 简化版 ATR 近似
    "Mean($high / $low - 1, 20)",
]

names = [
    "ema20_ema60_spread",
    "ema60_slope10",
    "dist_ema20",
    "breakout60_pos",
    "ret20",
    "ret60",
    "volume_ratio20",
    "volatility20",
    "atr_proxy20",
]
```

真实 ATR、ADX、RSI 如果 Qlib 表达式不方便，可以自己在数据预处理阶段算好，再写入自定义字段。

---

## 2. Label 要避免错位

如果你在 T 日收盘后得到信号，A 股通常 T+1 才能买入，所以不能用 T 日收盘价当作实际成交价。Qlib 的 Alpha158 文档也明确说明，中国股票在 T 日拿到收盘价后，股票 T+1 可买、T+2 可卖，因此 label 示例用的是 `Ref($close, -2)/Ref($close, -1)-1`，即 T+1 到 T+2 的变化，而不是 T 到 T+1。([Qlib Documentation][2])

如果你要预测未来 5 日收益，可以写成：

```python
labels = [
    "Ref($close, -6) / Ref($close, -1) - 1"
]
label_names = ["LABEL5D"]
```

含义是：

```text
T 日生成特征
T+1 买入
持有到 T+6
```

## 3. 策略实现：建议用自定义 Strategy

你的系统有：

```text
固定持仓状态
初始止损
时间止损
动态止盈
EMA60最终退出
按风险单位建仓
```

这类逻辑不是简单 TopK 排序策略，建议自定义策略。

Qlib 文档说明，用户可以继承 `BaseStrategy` 自定义策略，核心接口是 `generate_trade_decision`；也可以使用 `WeightStrategyBase`，只生成目标权重，由框架自动生成订单。([Qlib Documentation][3])

建议你分两版实现：

## 规则版

```text
BaseStrategy / WeightStrategyBase
直接实现：
- 市场过滤
- 行业过滤
- EMA候选
- trend_score排名
- 突破/回踩触发
- ATR止损
- EMA20/EMA60退出
```

## 模型版

```text
EMA 多头排列只做候选
LightGBM / CatBoost / MLP 预测未来5日/10日收益
模型分数 + trend_score 共同决定买入
策略层仍然负责止损、仓位和退出
```

Qlib 的 workflow 支持从数据加载、处理、切片，到模型训练、推理、信号分析和回测；`qrun` 可以按配置执行完整流程。([Qlib Documentation][4])

# 十四、回测设置一定要真实

A 股交易成本和成交限制非常重要。上交所 2026 年 1 月更新的收费表显示，A 股竞价交易经手费为成交金额的 0.00341% 双向收取。([SSE][5]) 财政部、税务总局公告也显示，自 2023 年 8 月 28 日起，证券交易印花税实施减半征收。([China Tax Policy Database][6])

所以 Qlib 回测里不要偷懒。至少设置：

```python
backtest_config = {
    "start_time": "2017-01-01",
    "end_time": "2025-12-31",
```

其中：

```text
deal_price = open：

close_cost > open_cost：
```

# 十五、实验顺序：不要一次性上全套

最好的方式不是直接写一个复杂系统，而是逐层加功能，做 ablation test。

建议顺序：

| 阶段 | 版本                       | 目的             |
| -- | ------------------------ | -------------- |
| 0  | 原始 EMA 多头排列策略            | baseline       |
| 1  | 加大盘过滤                    | 验证是否减少震荡市磨损    |
| 2  | 加行业过滤                    | 验证是否改善顺风交易     |
| 3  | 加 trend_score 排名         | 验证是否提升信号质量     |
| 4  | 入场拆成突破型/回踩型              | 比较哪种买点更适合大市值股  |
| 5  | 固定止损改 ATR 止损             | 验证亏损分布是否改善     |
| 6  | 加 EMA20 / ATR trailing   | 验证利润回吐是否下降     |
| 7  | 加风险单位仓位                  | 验证回撤是否更可控      |
| 8  | 加 LightGBM meta-labeling | 预测哪些 EMA 信号值得做 |

每个版本都看：

```text
年化收益
最大回撤
收益回撤比
超额收益
换手率
成本前收益 vs 成本后收益
胜率
平均盈利 / 平均亏损
持仓天数分布
剔除前5笔最大盈利后是否仍赚钱
不同市场阶段收益
突破型 vs 回踩型收益
```

尤其要看：

```text
成本前赚钱，成本后亏钱？
还是成本前也不赚钱？
```

如果成本前赚钱、成本后亏钱，说明主要问题是换手和执行。
如果成本前也不赚钱，说明信号本身没有优势，需要重做过滤和入场。

# 十六、最终整合结论

两套建议整合后，最终不是“多指标共振系统”，而是：

```text
市场状态决定能不能做；
行业强度决定去哪里做；
EMA 多头排列决定谁进入候选；
趋势质量评分决定谁更值得做；
突破/回踩触发决定什么时候做；
ATR 和结构止损决定错了亏多少；
EMA20 / ATR trailing 保护利润；
EMA60 保留最后一段肥尾。
```

一句话版本：

> **EMA 不再是买入信号，而是趋势候选信号；真正的交易决策由市场环境、行业强度、趋势质量、入场位置和风险收益比共同决定。**

我建议你优先实现这三个改动：

```text
第一：加大盘 + 行业过滤。
第二：EMA 多头排列只做候选，再用 trend_score 排名。
第三：退出改成 ATR/EMA20 保护利润 + EMA60 最终退出。
```

机器学习和复杂 Qlib 工作流可以后置。先把规则版跑稳，再让模型去做一件更窄、更有价值的事：

```text
判断这个 EMA 多头信号，是不是值得交易。
```
