# a_share

`a_share` 是一个面向中国 A 股市场的研究型仓库，用于持续沉淀与迭代以下内容：

- 与 A 股相关的数据处理、分析、回测与工具源码
- 对市场结构、行业轮动、风格切换、事件驱动等主题的研究笔记
- 策略想法、观察记录、问题清单与可验证假设

项目当前处于初始化阶段，优先目标不是快速堆功能，而是先建立一套清晰、可长期维护的知识与代码组织方式。

## 项目目标

本仓库希望解决三个问题：

1. 让研究结论可追溯，而不是停留在零散想法里。
2. 让策略与工具可复用，而不是每次重复搭建分析脚手架。
3. 让研究、代码、结论之间形成闭环，方便后续复盘与迭代。

## 仓库内容定位

后续内容可以围绕以下几个方向展开：

- `source code`：数据获取、清洗、指标计算、选股、回测、可视化等代码
- `research`：专题研究、市场观察、行业分析、因子分析、事件复盘
- `ideas`：策略草案、待验证假设、实验方向、问题列表

## 建议目录结构

考虑到后续会按研究主题拆分，每个主题目录都可以视为一个相对独立的子项目。它可以是以 Python 代码为主的工程，也可以是以 Notebook 为主的研究目录。

建议后续按下面的方式逐步整理：

```text
a_share/
├── README.md                          # 项目总说明
├── topics/                            # 按主题拆分的研究/工程目录
│   ├── market_breadth/                # 例：市场宽度研究
│   │   ├── README.md                  # 当前主题说明
│   │   ├── notes/                     # 研究笔记、结论、图表说明
│   │   ├── notebooks/                 # Notebook 研究文件
│   │   ├── src/                       # 若该主题包含可复用代码
│   │   ├── data/                      # 当前主题使用的数据
│   │   └── tests/                     # 当前主题相关测试
│   ├── sector_rotation/               # 例：行业轮动研究
│   │   ├── README.md
│   │   ├── notebooks/
│   │   └── data/
│   └── event_driven/                  # 例：事件驱动研究
├── shared/                            # 可跨主题复用的公共工具
│   ├── utils/
│   ├── data_sources/
│   └── templates/
└── archive/                           # 已暂停或归档的旧研究
```

这种结构的优点是：

- 每个主题目录都能独立演进，减少不同研究之间的相互干扰
- 更适合同时容纳不同工作流，例如纯 Notebook 研究、Python 工程化实现，或两者混合
- 便于后续把成熟主题单独抽出来做策略库、工具包或长期维护模块

## 适合研究的主题

可以从以下主题中选择优先级较高的方向开始：

- 市场宽度与情绪指标
- 风格切换：大盘 / 小盘、成长 / 价值
- 行业轮动与热点持续性
- 财报、分红、回购、减持等事件驱动影响
- 北向资金、融资融券、成交额结构变化
- 因子有效性与失效周期
- 择时信号与仓位管理
- 涨停板、连板、题材扩散等短线行为特征

## 推荐工作方式

为了保证仓库长期可用，建议遵循下面的原则：

- 每个主题目录尽量自包含，至少有自己的说明文档、数据说明和研究输出
- 每个研究主题尽量有明确问题定义、数据范围、方法、结论与后续问题
- 每个策略想法都尽量转化为可验证假设，而不是停留在主观判断
- 如果某个主题已经开始工程化，再在该主题目录内继续拆分数据处理、信号生成、回测与可视化逻辑
- 重要结论尽量附带图表、样本区间、边界条件与失败场景
- 研究结论与实现代码相互链接，方便从结论回溯到实现
